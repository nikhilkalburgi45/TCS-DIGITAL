# 1. Fibonacci (Recursion Only)

## Problem Statement (LeetCode Style)

Given an integer `n`, return the **n-th Fibonacci number**.

Fibonacci series:

```
F(0) = 0
F(1) = 1
F(n) = F(n-1) + F(n-2)
```

---

## 📥 Input

```
n = 5
```

## 📤 Output

```
5
```

---

## 🧠 Core Intuition (Simple Language)

Break the problem into **smaller same problems**.

To find:

```
F(5)
```

You need:

```
F(4) + F(3)
```

To find `F(4)`:

```
F(3) + F(2)
```

This keeps breaking until:

```
F(1) = 1
F(0) = 0
```

👉 This is **recursion = function calling itself**

---

## ❌ Brutal Truth About Recursion Here

- Repeats same work again and again
- Very slow for big `n`
- Time Complexity = **O(2^n)** (exponential)

Example:

```
F(5) calls F(3) multiple times
```

This is wasteful — DP will fix this later.

---

## 🧩 Recursive Tree (Understand This Properly)

For `n = 5`:

```
F(5)
├── F(4)
│   ├── F(3)
│   │   ├── F(2)
│   │   │   ├── F(1)
│   │   │   └── F(0)
│   │   └── F(1)
│   └── F(2)
└── F(3)
```

👉 Same subproblems repeated → BAD

---

## 🪜 Step-by-Step Algorithm

1. If `n == 0` → return 0
2. If `n == 1` → return 1
3. Otherwise:

   ```
   return fib(n-1) + fib(n-2)
   ```

---

## 💻 C++ Code (Clean + Commented)

```cpp
#include <iostream>
using namespace std;

// Recursive function
int fib(int n) {
    // Base case
    if (n == 0) return 0;
    if (n == 1) return 1;

    // Recursive call
    return fib(n - 1) + fib(n - 2);
}

int main() {
    int n;
    cin >> n;

    cout << fib(n);

    return 0;
}
```

---

## 🔍 Dry Run

Input:

```
n = 4
```

Execution:

```
fib(4)
= fib(3) + fib(2)
= (fib(2) + fib(1)) + (fib(1) + fib(0))
= ((1 + 1) + 1) + (1 + 0)
= 3
```

---

## ⚠️ Edge Cases

| Case  | Output                |
| ----- | --------------------- |
| n = 0 | 0                     |
| n = 1 | 1                     |
| n < 0 | Invalid (not defined) |

---

## ⏱ Complexity

| Type  | Value                  |
| ----- | ---------------------- |
| Time  | O(2^n) ❌              |
| Space | O(n) (recursion stack) |

---

## 📌 Key Learning

- Recursion = breaking into smaller same problems
- Always define **base case first**
- Works but inefficient due to repeated calls
- This inefficiency leads to **Dynamic Programming**

---

## 🚨 Common Mistakes

- Forgetting base case → infinite recursion
- Using `n <= 1` but returning wrong value
- Not understanding recursion tree

---

## 🧠 Real Insight (Important)

Fibonacci is not about numbers — it’s about learning:

- How recursion expands
- Why overlapping subproblems are bad
- Why DP exists
